Introduction

  • Item Name : Dynamic Web Check
  • Item Version : v 1.0
  • Author : Montek Singh Kundan
  • Support Forum : None right now
  • License : MIT License

Dynamic web check is a framework built using selenium and pytest to simplify test automations.
It has built in pre-written functions, which use selenium as its backbone, all the test cases run with a pre-customized pytest conftest file.

This documentation is to help you regarding how to use the functions and how to customize each step. Please go through the documentation carefully to understand how this framework is made and how to work with it and customize this properly. Basic PYTHON knowledge is required to use/modify this framework. You may learn basics here, here and here.

Requirements

Be careful while modifying the framework. If required packages are not installed, the framework may break and output error messages.
No support is provided for faulty customization.

Getting Started Back to top

After un-zipping the file, the default folder is set to smoke, the default folder can be changed from pytest.ini file. To run the code open the terminal in the root path of the framework folder.

Be careful you need a env variables in your system to generate report in particular location, by default reports will be generated in the root directory of the project.

Contents for env variables

LOG_PATH = path
REPORT_PATH = path

Deployment

pytest

For Windows

python -m pytest

To run a test for particular file

pytest 'filename'

To run a test for particular file, For Windows

python -m pytest 'filename'

Displaying website URLBack to top

.display_url

This command prints the url of the main url sent to the webdriver.

from FUNCTIONS.helper import Checker
    def test(browser):
        run = Checker("url", browser)
        run.display_url()

.return_url

This command returns the url of the main url sent to the webdriver.

from FUNCTIONS.helper import Checker
    def test(browser):
        run = Checker("url", browser)
        website_url = run.return_url()
        print(f"Website url is: {website_url}")

.valid_url

This command checks and returns the url if its valid, if it is not valid (in case of a wrong website url, or a non working localhost url) it asserts false.

from FUNCTIONS.helper import Checker
    def test(browser):
        run = Checker("url", browser)
        run.valid_url("http://wrongurl.com")

.current_tab_url

This command returns the url of the current website in the current selected tab.

from FUNCTIONS.helper import Checker
    def test(browser):
        run = Checker("url", browser)
        current_tab_url = run.current_tab_url()
        print(f"New URL is: {current_tab_url}")

Function .input Back to top

This function takes an input and types it into the path given to the element.

With XPATH

There are many ways of selecting an element, the most effective way is using XPATH.

from FUNCTIONS.helper import Checker
    def test(browser):
        run = Checker("url", browser)
        run.input("xpath", "//path", "type", "Sample Value")

With CSS

If XPATH is not what works great for the work, CSS selector is also an option.

from FUNCTIONS.helper import Checker
    def test(browser):
        run = Checker("url", browser)
        run.input("css", "//path", "type", "Sample Value")

Other selectors which can be used:

  • name
  • link_text
  • partial_link_text
  • tag_name
  • class_name
