Introduction

  • Item Name : Dynamic Web Check
  • Item Version : v 1.0
  • Author : Montek Singh Kundan
  • Support Forum : None right now
  • License : MIT License

Dynamic web check is a framework built using selenium and pytest to simplify test automations.
It has built in pre-written functions, which use selenium as its backbone, all the test cases run with a pre-customized pytest conftest file.

This documentation is to help you regarding how to use the functions and how to customize each step. Please go through the documentation carefully to understand how this framework is made and how to work with it and customize this properly. Basic PYTHON knowledge is required to use/modify this framework. You may learn basics here, here and here.

Requirements

Be careful while modifying the framework. If required packages are not installed, the framework may break and output error messages.
No support is provided for faulty customization.

Getting Started Back to top

After un-zipping the file, the default folder is set to smoke, the default folder can be changed from pytest.ini file. To run the code open the terminal in the root path of the framework folder.

Deployment

pytest

.display_url

This command prints the url of the main url sent to the webdriver.

from FUNCTIONS.helper import Checker
    def test(setup):
        run = Checker("url", setup)
        run.display_url()

.return_url

This command returns the url of the main url sent to the webdriver.

from FUNCTIONS.helper import Checker
    def test(setup):
        run = Checker("url", setup)
        run.display_url()
