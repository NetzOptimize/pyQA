Dynamic web check is a framework built using selenium and pytest to simplify test automations.
It has built in pre-written functions, which use selenium as its backbone, all the test cases run with a pre-customized pytest conftest file.

This documentation is to help you regarding how to use the functions and how to customize each step. Please go through the documentation carefully to understand how this framework is made and how to work with it and customize this properly. Basic PYTHON knowledge is required to use/modify this framework. You may learn basics here, here and here.

  1. pytest==7.1.2

  5. webdriver_manager==3.8.3
  6. pytest-parallel==0.1.1
  7. pytest-excel==1.5.0
  8. pytest-instafail==0.4.2
  9. markdown==3.4.1
  10. md-mermaid==0.1.1
  11. py==1.11.0
  12. openpyxl==3.0.10
  13. Jinja2==3.1.2
  14. requests~=2.28.1

Be careful while modifying the framework. If required packages are not installed, the framework may break and output error messages.
No support is provided for faulty customization.

Getting Started Back to top

After un-zipping the file, the default folder is set to smoke, the default folder can be changed from pytest.ini file. To run the code open the terminal in the root path of the framework folder.

Be careful you need a env variables in your system to generate report in particular location, by default reports will be generated in the root directory of the project.

Contents for env variables

LOG_PATH = path
REPORT_PATH = path

Built-in Functions

There are many built-in functions provided for you to use. These functions use selenium as the backbone and simplifies the usage of selenium functions.
Using these functions allows you to integrate them with pytest, allowing you to run multiple test cases and generating a report file. Documentation to understand the implementation of the functions can be accessed through this link Built-in Functions.

How to Customize pytest.ini Back to top

pytest.ini file is kept at the root of the framework directory..
Please read more about pytest.ini configuration here. Configuration

Defaults

[pytest]
markers =
    smoke: tests that check is functions work properly.
    regression: tests that are to be run finally.
testpaths =
    smoke
addopts = --verbose --workers 2 --tests-per-worker auto --instafail

Exception for Windows

Currently for Windows a pytest plugin pytest-parallel does not work. Remove the arguments --workers 2 --tests-per-worker auto from the addopts configuration, it should look like this now:

addopts = --verbose --instafail

Necessary PluginsBack to top

webdriver-manager

It simplifies management of binary drivers for different browsers.

pytest-html

pytest-html is a plugin for pytest that generates a HTML report for test results.

pytest-parallel

This plugin makes it possible to run tests quickly using multiprocessing (parallelism) and multithreading (concurrency).

These plugins are necessary because they controll the workflow of this framework,webdriver-manager helps in automatically getting the binaries of the webdrivers . pytest-html generates the html report of the test runs. pytest-parallel This plugin helps in running multiple tests parallel, which reduces test run time.

Creating New Folders and TestsBack to top

Pytest allows us to provide a custom testpath to run all test from, this means that when we run the command pytest (or python -m pytest for window users), all tests provided in the set testpath will run without specifying the pathname for testcases. Also there is a set rule to name test files.

pytest.ini

  • Step 1 - The default folder is final_test, you can change it too.
  • Step 2 - Run the command to start testing all the files in the default set folder.

Folders

Folders must have test_ at the starting, and folders insided test folders can be named anything.

  • Step 1 - Folders inside test folders must have __init__.py file for pytest to recognize test case folders.

Test files

Test case files must have test_ at the starting.

  • Step 1 - Test case files should have test cases made from the built-in functions.
